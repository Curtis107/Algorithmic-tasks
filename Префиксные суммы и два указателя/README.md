# Префиксные суммы и два указателя

# Задача 1 (Префиксная сумма)

Дана последовательность чисел. Нужно вычислить последовательность ее префиксных сумм

### **Формат ввода**

В первой строке передается число *n* - количество чисел в последовательности (1 <= *n* <= 10<sup>3</sup>)

В следующей строке передается *n* чисел. Каждое число не превосходит по модулю 10<sup>6</sup>

### **Формат вывода**

В ответе выведите последовательность префиксных сумм (*n* чисел)

### Пример
```
5
1 2 3 4 5  ->  1 3 6 10 15
```

## Объяснение решения

Задача нужна исключительно для того, чтобы познакомиться с тем, что такое префиксная сумма. Массив префиксных сумм - это массив длины *n*+1, в котором в ячейке с индексом 0 стоит 0, а в *i*-ых (*i* != 0) - сумма первых *i* элементов последовательности. Префиксная сумма легко помогает дать ответ на вопрос: "чему равна сумма элементов на отрезке [*L*; *R*]" - она равна *массив*[*R* + 1] - *массив*[*L*]

```python
n = int(input())
nums = list(map(int, input().split()))
prefix_sums = [0] * (n + 1)
for i in range(1, n + 1):
    prefix_sums[i] = prefix_sums[i - 1] + nums[i - 1]
    print(prefix_sums[i], end=' ')
```

# Задача 2 (Сумма на отрезках)

Дана последовательность чисел из *N* элементов. Далее идут *q* запросов в виде: *L* *R*, для которых нужно ответить чему равна сумма на отрезке [*L*; *R*]

*Примечание: запросы составляются самостоятельно по определенному алгоритму:*

*1. Генерируется последовательность чисел X<sub>0</sub>, ..., X<sub>2q-1</sub> по формуле: X<sub>i</sub> = (11173 * X<sub>i-1</sub> + 1)mod1000000007*

*2. Генерируются числа L<sub>0</sub>, ..., L<sub>q-1</sub> и R<sub>0</sub>, ..., R<sub>q-1</sub> по формулам: L<sub>0</sub> = min(X<sub>0</sub>modN, X<sub>1</sub>modN), R<sub>0</sub> = max(X<sub>0</sub>modN, X<sub>1</sub>modN); L<sub>1</sub> = min(X<sub>2</sub>modN, X<sub>3</sub>modN), R<sub>2</sub> = max(X<sub>2</sub>modN, X<sub>3</sub>modN) и т.д*

### **Формат ввода**

В первой строке передается число *N* - количество чисел в последовательности (1 <= *N* <= 10<sup>5</sup>)

В следующей строке передается *N* чисел. Каждое число не превосходит по модулю 10<sup>9</sup>

В третьей строке передается число *q* - количество запросов (1 <= *q* <= 10000)

В четвертой строке передается число *X<sub>0</sub>* - начальное число для генерации другиз чисел (0 <= *X<sub>0</sub>* <= 10<sup>9</sup>)

### **Формат вывода**

В ответе выведите одно число - остаток от деления суммы всех результатов запросов на 1000000007

### Пример
```
5
1 2 3 4 5   ->  27
3
123

7
34 10 49 32 28 15 70  ->  492
5
0
```

## Объяснение решения

Здесь воспользуемся тем же массивом префиксных сумм. Главное - правильно прочитать условие и написать генератор чисел. Кроме того, при подсчете суммы всех запросов, каждый раз пользуемся операцией *%*, так как если случится переполнение - то ответ может выйти совершенно другой

```python
def get_sequence(x0):
  x1 = (11173 * x0 + 1) % 1000000007
  return x1

def get_l_r(x0, x1, n):
  l0 = min(x0 % n, x1 % n)
  r0 = max(x0 % n, x1 % n)
  return l0, r0

n = int(input())
nums = list(map(int, input().split()))
q = int(input())
x0 = int(input())
prefix_sums = [0] * (n + 1)
for i in range(1, n + 1):
    prefix_sums[i] = prefix_sums[i - 1] + nums[i - 1]

ans = 0
x1 = get_sequence(x0)
for i in range(q):
    l, r = get_l_r(x0, x1, n)
    ans = (ans + prefix_sums[r + 1] - prefix_sums[l]) % 1000000007
    x0 = get_sequence(x1)
    x1 = get_sequence(x0)
print(ans % 1000000007)
```

# Задача 3 (Максимальная сумма)

Дана последовательность из *n* чисел. Нужно вычислить максимальную сумму, среди всех отрезков [*L*; *R*]

### **Формат ввода**

В первой строке передается число *n* - количество чисел в последовательности (1 <= *n* <= 3*10<sup>5</sup>)

В следующей строке передается *n* чисел. Каждое число не превосходит по модулю 10<sup>9</sup>

### **Формат вывода**

В ответе выведите максимальную сумму среди всех отрезков

### Пример
```
4
1 2 3 4    ->    10

4
5 4 -10 4    ->    9

4
5 -2 18 1    ->    22
```

## Объяснение решения

Максимальная сумма может быть как положительной, так и отрицательной (и еще 0, но отнесем его в случай отрицательных). С положительной все понятно - это первое, что нам приходит в голову. Но отрицательной она может быть только в том случае, если ни на каком подотрезке нет положительной суммы, значит, ответ это наибольший элемент последовательности. Но в большинстве случаев нам предстоит решить задачу по поиску положительной суммы. Тут есть несколько подходов

1. Давайте суммировать элементы последовательности, пока их сумма положительна. Если она стала отрицательной, то мы сбрасываем текцщую префиксную сумму и начинаем заново с места, где остановились. Логика довольно проста: зачем нам добавлять в накопившуюся сумму еще элементы, если она уже отрицательна, проще взять следующий элемент, ведь он будет больше, чем вся префиксная сумма.

```python
n = int(input())
nums = list(map(int, input().split()))
mx_sum = nums[0]
now_sum = 0
for i in range(n):
    now_sum += nums[i]
    mx_sum = max(mx_sum, now_sum)
    if now_sum < 0:
        now_sum = 0
print(mx_sum)
```

2. Второй подход заключается в нахождении текущей минимальной префиксной суммы. Давайте рассмотрим 2-ой пример: массив префиксных сумм выгядит так [0, 5, 9, -1, 3]. На каждом шаге мы будем высчитывать текущую префиксную сумму, а также запоминать, какая префиксная сумма была минимальной к текущему моменту. Ответом будет максимальная разница между текущей префиксной суммой и последней минимальной. То есть для в примере на 1 шаге минимальная - 0, текущая - 5, разница - 5; на втором - 0 9 9; на 3-ем -1 -1 0; на 4-ом -1 3 4. Ответ: 9

```python
n = int(input())
nums = list(map(int, input().split()))
mn_sum = 0
now_sum = 0
mx_sum = nums[0]
for i in range(n):
    now_sum += nums[i]
    mx_sum = max(mx_sum, now_sum - mn_sum)
    mn_sum = min(mn_sum, now_sum)
print(mx_sum)
```

# Задача 4 (Количество сумм)

Дана последовательность из *n* чисел. Нужно вычислить количество отрезков, на которых сумма элементов равна некоторому числу *k*

### **Формат ввода**

В первой строке передается два числа: *n* - количество чисел в последовательности и *k* (1 <= *n* <= 10<sup>5</sup>; 1 <= *k* <= 10<sup>9</sup>)

В следующей строке передается *n* чисел. Каждое число натуральное и не превосходит 999

### **Формат вывода**

В ответе выведите количество отрезков, на которых сумма равна *k*

### Пример
```
5 17
17 7 10 7 10    ->    4

5 10
1 2 3 4 1    ->    2
```

## Объяснение решения

Пусть массив префиксных сумм - pref. Нам нужно найти такие отрезки, что *pref*[*R*] - *pref*[*L*] = *k*, значит, *pref*[*R*] - *k* = *pref*[*L*]. Пусть *R* - это текущая префиксная сумма, тогда нам нужно знать, была ли такая сумма *pref*[*R*] - *k*. Для этого воспользуемся словарем, на каждой итерации будем подсчитывать количество сумм, равных *pref*[*R*] - *k*

```python
n, k = map(int, input().split())
nums = list(map(int, input().split()))
sums = {0: 1}
prefix_sum = 0
count = 0
for i in range(n):
    prefix_sum += nums[i]
    if prefix_sum - k in sums:
        count += sums[prefix_sum - k]
    if prefix_sum not in sums:
        sums.update({prefix_sum: 0})
    sums[prefix_sum] += 1
print(count)
```

# Задача 5 (Тройные произведения)

Дана последовательность из *n* чисел. Нужно вычислить сумму всех тройных произведений (&sum;<sub>1<=*i*<*j*<*k*<=*n*</sub> *a<sub>i</sub>a<sub>j</sub>a<sub>k</sub>*) чисел по модулю 1000000007 

### **Формат ввода**

В первой строке передается число *n* - количество чисел в последовательности (3 <= *n* <= 10<sup>6</sup>)

В следующей строке передается *n* чисел. Каждое число больше 0 и не превосходит 10<sup>6</sup>

### **Формат вывода**

В ответе укажите сумму всех тройных произведений по модулю 1000000007

### Пример
```
3
1 2 3    ->    6

4
1 2 3 4    ->    50

4
0 5 6 7    ->    210
```

## Объяснение решения

Самое наивное решение будет работать за O(n<sup>3</sup>), но можно добиться результата лучше - за O(n). Есть 2 подхода:

1. Алгоритмический:

Допустим нам даны фиксированные *a<sub>j</sub>* и *a<sub>k</sub>*. Тогда все тройные произведения считаются одним циклом - перебором значений *a<sub>i</sub>* (1 <= *i* < *j*). Это так же можно представить в виде суммы чисел *a*<sub>1</sub>, ..., *a*<sub>*j*-1</sub>, умноженных на *a<sub>j</sub>a<sub>k</sub>*, то есть префиксная сумма первых *j*-1 элементов на *a<sub>j</sub>a<sub>k</sub>*). Отсюда можем добиться решения за O(n<sup>2</sup>) - перебирать параметры *j* и *k*, а префиксные суммы вычислить заранее.

Теперь пусть у нас зафиксировано только *a<sub>j</sub>*. Пользуемся предыдущей идеей: *a<sub>i</sub>*-ые будем подсчитывать через префиксные суммы, тогда для каждого *k*: *преф.сумма * a<sub>j</sub> * a<sub>j+1</sub>* + *преф.сумма * a<sub>j</sub> * a<sub>j+2</sub>* + ... + *преф.сумма * a<sub>j</sub> * a<sub>n</sub>*. Представим иначе: *преф.сумма * a<sub>j</sub> * (a<sub>j+1</sub> + ... + a<sub>n</sub>)*, а это суффиксная сумма - сумма элементов, начиная с конца массива. Тогда, нам нужно перебирать значение *j* и умножать на префиксную и суффиксную суммы

```python
n = int(input())
nums = list(map(int, input().split()))
prefix_sums = [0] * (n + 1)
for i in range(1, n + 1):
    prefix_sums[i] = prefix_sums[i - 1] + nums[i - 1]

ans = 0
for i in range(1, n + 1):
    ans = (ans + nums[i - 1] * prefix_sums[i - 1] * (prefix_sums[n] - prefix_sums[i])) % 1000000007
print(ans % 1000000007)
```

2. Математический:

Рассмотрим: *(a+b+c)<sup>3</sup> = a<sup>3</sup> + b<sup>3</sup> + c<sup>3</sup> + 3ac<sup>2</sup> + 3ab<sup>2</sup> + 3a<sup>2</sup>b + 3a<sup>2</sup>c + 3b<sup>2</sup>c + 3bc<sup>2</sup> + 6abc*

Это можно представить иначе: *(a+b+c)<sup>3</sup> = a<sup>3</sup> + b<sup>3</sup> + c<sup>3</sup> + 3(ab+bc+ac)(a+b+c) - 3abc* - отсюда выразим abc:

*abc = 1/3 * (a<sup>3</sup> + b<sup>3</sup> + c<sup>3</sup> + 3(ab+bc+ac)(a+b+c) - (a+b+c)<sup>3</sup>)*. Если у нас не 3 числа, а больше, то можем преобразовать наши уравнения:

*&sum; a<sub>i</sub>a<sub>j</sub>a<sub>k</sub> = 1/3 * (&sum; (a<sub>i</sub>)<sup>3</sup> + 3&sum;a<sub>i</sub>a<sub>j</sub>&sum;a<sub>i</sub> - (&sum;a<sub>i</sub>)<sup>3</sup>)*. По такому же принципу преобразуем сумму попарных произведений:

*(a+b)<sup>2</sup> = a<sup>2</sup> + b<sup>2</sup> + 2ab*

*ab = 1/2 * ((a+b)<sup>2</sup> - (a<sup>2</sup> + b<sup>2</sup>)). При большем количестве чисел: &sum;a<sub>i</sub>a<sub>j</sub> = 1/2 * ((&sum;a<sub>i</sub>)<sup>2</sup> - &sum;(a<sub>i</sub>)<sup>2</sup>)

Теперь, имея все формулы, сможем посчитать исходную сумму

```python
n = int(input())
nums = list(map(int, input().split()))
s1 = sum(i for i in nums)
s2 = sum(i * i for i in nums)
s3 = sum(i * i * i for i in nums)
print(((s3 + 3 * ((s1**2 - s2) // 2) * s1 - s1**3) // 3) % 1000000007)
```

# Задача 6 (Набор одежды)

Даны два списка цветов маек и штанов. Глеб хочет подобрать майку и штаны таким образом, чтобы разница в цвете между ними была минимальная. Цвета в списках упорядочены по неубыванию

### **Формат ввода**

В первой строке передается число *n* - размер списка цветов маек (1 <= *n* <= 100000)

В следующей строке передается список цветов маек

В третьей строке передается число *m* - размер списка цветов штанов (1 <= *m* <= 100000)

В следующей строке передается список цветов штанов

### **Формат вывода**

В ответе укажите два числа через пробел - цвет майки и цвет штанов, которые Глеб должен выбрать. Если вариантов несколько - выведите любой

### Пример

```
2
3 4
3        ->    3 3
1 2 3

2
4 5
3        ->    4 3
1 2 3

5
1 2 3 4 5
5            ->    1 1
1 2 3 4 5
```

## Объяснение решения

Здесь воспользуемся методом двух указателей: одним будем проходиться по массиву маек, другим - по массиву штанов. Будем сравнивать абсолютную разницу между цветами, потому что в общем случае мы не знаем, какой цвет численно больше. Двигать указатели будем по такому принципу: если цвет текущей майки меньше, чем цвет текущих штанов, то увеличиваем указатель маек, иначе - указатель штанов. Таким образом мы стремимся найти элементы, которые отличаются друг от друга меньше всех

```python
n = int(input())
shirts = list(map(int, input().split()))
m = int(input())
jeans = list(map(int, input().split()))
shirts_color, jeans_color = 0, 0
mn_diff = 10**7
l, r = shirts[0], jeans[0]
while shirts_color < n and jeans_color < m:
    if mn_diff > abs(shirts[shirts_color] - jeans[jeans_color]):
        mn_diff = abs(shirts[shirts_color] - jeans[jeans_color])
        l, r = shirts[shirts_color], jeans[jeans_color]

    if shirts[shirts_color] < jeans[jeans_color]:
        shirts_color += 1
    else:
        jeans_color += 1

print(l, r)
```


# Задача 7 (Столбы в городе)

Вдоль прямой улицы установлены столбы. Дан массив, в котором указана координата каждого столба от начала улицы. Нужно посчитать количество пар столбов, которые удалены друг от друга более чем на *r* метров

### **Формат ввода**

В первой строке передается два числа *n* и *r* - количество чисел в последовательности (2 <= *n* <= 300000) и необходимое расстояние между столбами соответственно (1 <= *r* <= 10<sup>9</sup>)

В следующей строке передается *n* чисел. Каждое число не превосходит 10<sup>9</sup>

### **Формат вывода**

В ответе укажите количество пар столбов, удаленных друг от друга на расстоянии более *r* метров

### Пример
```
4 4
1 3 5 8    ->    2
```

## Объяснение решения

Заведем 2 указателя. Для каждого левого указателя будем двигать правый, пока расстояние между столбами меньше или равно *r*. Как только нашли первый подходящий столб, то к ответу прибавляем данную пару и все столбы, стоящие после того, на котором стоит правый указатель, т.к они будут тоже на расстоянии, большем чем *r*. Потом двигаем левый, а правый будем двигать с того же места, где он остановился

```python
n, r = map(int, input().split())
monuments = list(map(int, input().split()))
right, count = 0, 0
for left in range(n):
    while right < n and monuments[right] - monuments[left] <= r:
        right += 1

    if right != n and monuments[right] - monuments[left] > r:
        count += n - right

print(count)
```

# Задача 8 (Победила Дружба)

Вася и Маша соревнуются в поедании сырков, которые расположены на *n* столах. Вася начинает со стола номер 1, Маша - с номера *n*. Количество съеденных сырков должно отличаться как можно меньше. Укажите минимальную разницу съеденных сырков и номера столов, где достигается минимум

### **Формат ввода**

В первой строке передается число *n* - количество столов (2 <= *n* <= 10<sup>6</sup>)

В следующей строке передается *n* чисел. Каждое число не превосходит 10<sup>9</sup>

### **Формат вывода**

В ответе укажите минимальную разность съденных сырков и номера столов, при которых это достигается

### Пример
```
5
5 1 1 1 1    ->    1 1 2

4
1 2 3 4    ->    1 2 4
```

## Объяснение решения

Один указатель поставим на начало массива, а второй в конец. Будем накапливать суммы съеденных сырков Васей и Машей. Если сумма съеденных сырков у Васи превосходит сумму Маши, то двигаем указатель Маши, иначе - Васи. Продолжаем, пока укзаатели не встретятся

```python
n = int(input())
arr = list(map(int, input().split()))
SDv, SDm = 0, n - 1
sum_left, sum_right = arr[SDv], arr[SDm]
left, right = 0, n - 1
mn_diff = 10**9
while left < right:
    if mn_diff > abs(sum_left - sum_right):
        mn_diff = abs(sum_left - sum_right)
        SDv, SDm = left, right

    if sum_left < sum_right:
        left += 1
        if left < right:
            sum_left += arr[left]
    else:
        right -= 1
        if left < right:
            sum_right += arr[right]

print(mn_diff, SDv + 1, SDm + 1)
```

# Задача 9 (Красота превыше всего)

Вдоль аллеи посажены деревья различных сортов. Нужно найти подотрезок минимальной длины, в котором встречается дерево каждого сорта хотя бы один раз. Если ответов несколько - выведите любой

### **Формат ввода**

В первой строке передается два числа *n* и *k* - количество деревьев и количество сортов соответственно (1 <= *n*, *k* <= 250000)

В следующей строке передается список сортов деревьев

### **Формат вывода**

В ответе укажите координаты начала и конца подотрезка минимальной длины

### Пример
```
5 3
1 2 1 3 2    ->    2 4

6 4
2 4 2 3 3 1    ->    2 6

10 3
1 1 1 1 3 2 2 2 2 1    ->    4 6
```

## Объяснение решения

Чтобы поддерживать количество сортов на текущем подотрезке, заведем словарь, в котором ключом будет сорт, а значением его количество на подотрезке. Будем двигаться по массиву сортов (левый указаьель) и для каждого будем смещать правый указатель, чтобы подсчитывать количество уникальных сортов. Как только оно достигло нужного числа *k*, то будем пытаться сократить левую границу отрезка, двигая левый указатель

```python
n, k = map(int, input().split())
trees = list(map(int, input().split()))
d = {}
for i in trees:
    d[i] = d.get(i, 0)

left, right = 0, 0
used = 0
l, r = 0, 250000

while left < n:
    while right < n and used < k:
        if d[trees[right]] == 0:
            used += 1
        d[trees[right]] += 1
        right += 1

    if r - l > (right - 1) - left and used == k:
        l, r = left, right - 1

    d[trees[left]] -= 1
    if d[trees[left]] == 0:
        used -= 1
    left += 1

print(l + 1, r + 1)
```
